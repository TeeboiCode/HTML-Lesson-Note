Lesson 01

Course Introduction

What is HTML and why does it matter?

What is HTML?

HTML stands for HyperText Markup Language. It is the foundational language of the web. Every webpage you have ever visited — from Google to YouTube to Wikipedia — is built with HTML at its core.

HTML is not a programming language. It is a markup language, meaning it describes the structure and meaning of content. Think of HTML as the skeleton of a webpage, CSS as the skin and clothing, and JavaScript as the muscles that make it move.

A Brief History

HTML was created by Tim Berners-Lee in 1991. Since then it has evolved through several versions — HTML 2.0, HTML 4.01, XHTML — and the current living standard is HTML5, maintained by the WHATWG.

How the Web Works

When you type a URL into your browser:

  1. Your browser sends a request to a web server.
  2. The server responds with an HTML file.
  3. The browser reads (parses) the HTML and displays it as a page.
💡
Good to know

You don't need an internet connection to practice HTML. You can create .html files on your computer and open them directly in your browser!

Tools You Need

To write HTML, all you need is:

  • A text editor — VS Code (recommended), Notepad++, Sublime Text, or even plain Notepad.
  • A web browser — Chrome, Firefox, Edge, or Safari.

Your First HTML File

Create a file called index.html and write the following: 

<!DOCTYPE html>
<html>
  <head>
    <title>My First Page</title>
  </head>
  <body>
    Hello, World!
  </body>
</html>

Save the file and open it in your browser. You should see "Hello, World!" displayed!

Structure of an HTML Page

Anatomy of every HTML document

The Boilerplate

Every HTML page follows a standard structure called the boilerplate. Here is the full structure with explanations: 

<!-- 1. DOCTYPE declaration -->
<!DOCTYPE html>

<!-- 2. Root element -->
<html lang="en">

  <!-- 3. Head section (invisible to user) -->
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
  </head>

  <!-- 4. Body section (visible to user) -->
  <body>
    All visible content goes here
  </body>

</html>

Breaking It Down

1. DOCTYPE Declaration

<!DOCTYPE html> — This tells the browser that this document is HTML5. It must be the very first line of every HTML file. It is not a tag; it is a declaration.

2. The <html> Element

This is the root element — all other elements are nested inside it. The lang="en" attribute tells browsers and screen readers what language the page is in.

3. The <head> Element

The <head> section contains metadata — information about the document that is NOT displayed on screen. This includes the page title, character encoding, viewport settings, and links to CSS stylesheets.

4. The <body> Element

The <body> section contains everything visible on the page — text, images, links, forms, etc.

⚠️
Common Mistake

Never put visible content (text, images) inside the <head>. It belongs in the <body>. The browser will often display it anyway, but this is incorrect HTML.

HTML Comments

Comments are notes for developers that the browser ignores entirely. 

<!-- This is a comment. The browser will not display this. -->

<!-- 
  Comments can also
  span multiple lines
-->

Lesson 03

HTML Elements, Tags & Attributes

The building blocks of every webpage

What is an HTML Element?

An HTML element consists of a start tag, content, and an end tag: 

<!-- Anatomy of an element -->
<p>This is a paragraph.</p>
↑         ↑                    ↑
Start tag  Content           End tag

Opening vs Closing Tags

<!-- Opening tag -->
<p>

<!-- Closing tag (note the forward slash) -->
</p>

<!-- Self-closing / void elements (no closing tag needed) -->
<br>
<hr>
<img src="photo.jpg">
<input type="text">

Nesting Elements

Elements can be placed inside other elements. This is called nesting. Always close inner elements before outer ones. 

<!-- ✅ Correct nesting -->
<p>This is <strong>bold text</strong> inside a paragraph.</p>

<!-- ❌ Wrong nesting (overlapping tags) -->
<p>This is <strong>wrong</p></strong>

HTML Attributes

Attributes provide additional information about an element. They always go in the opening tag and follow a name="value" pattern. 

<!-- Attribute syntax -->
<tagname attribute="value">

<!-- Multiple attributes -->
<a href="https://example.com" target="_blank">Click here</a>

<!-- Boolean attributes (no value needed) -->
<input type="checkbox" checked>
<button disabled>Cannot Click</button>

Common Global Attributes

Attribute Description Example
id Unique identifier for one element id="main-title"
class Groups elements for CSS styling class="card"
style Inline CSS styling style="color: red;"
title Tooltip shown on hover title="More info"
lang Language of the element's content lang="fr"
hidden Hides the element from view hidden
✅
Best Practice

Always use lowercase for tag names and attribute names. Always wrap attribute values in double quotes. These are the modern HTML5 standards.

Lesson 04

HTML Meta Tags

Invisible data that powers your page

What Are Meta Tags?

Meta tags live inside the <head> and provide metadata — information about the page that browsers, search engines, and social media platforms use. Users don't see them directly, but they have a huge impact on how the page behaves and ranks.

Essential Meta Tags

<head>
  <!-- Character Encoding -->
  <meta charset="UTF-8">

  <!-- Responsive viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Page description (shown in Google results) -->
  <meta name="description" content="Learn HTML from scratch in this beginner course.">

  <!-- Author -->
  <meta name="author" content="Jane Doe">

  <!-- Keywords (less important today, but still used) -->
  <meta name="keywords" content="HTML, web, beginner, course">

  <!-- Refresh the page every 30 seconds -->
  <meta http-equiv="refresh" content="30">
</head>

Open Graph Meta Tags

Open Graph tags control how your page looks when shared on social media (Facebook, Twitter, WhatsApp, etc.). 

<!-- Open Graph (social media preview) -->
<meta property="og:title" content="Learn HTML – Beginner Course">
<meta property="og:description" content="A complete HTML course for beginners.">
<meta property="og:image" content="https://example.com/thumbnail.jpg">
<meta property="og:url" content="https://example.com">
Meta Tag Purpose
charset="UTF-8" Supports all characters including special symbols and emojis
name="viewport" Makes the page mobile-responsive
name="description" Search engine result snippet (keep under 160 characters)
name="author" Credits the page author
name="robots" Controls search engine indexing behavior
🔍
SEO Tip

The description meta tag is one of the most important for SEO (Search Engine Optimisation). Write it carefully — it's what users see under your page title in Google search results!

Lesson 05

HTML Paragraphs & Headings

Structuring text content on the page

Heading Tags (h1–h6)

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important). Headings form a hierarchy — like an outline for your content. 

<h1>Main Page Title (only one per page)</h1>
<h2>Section Heading</h2>
<h3>Sub-section Heading</h3>
<h4>Sub-sub-section Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Paragraph Tags

<!-- Basic paragraph -->
<p>This is a paragraph. The browser automatically adds space before and after.</p>

<!-- Line break inside a paragraph -->
<p>First line of text.<br>
Second line within the same paragraph.</p>

<!-- Horizontal rule (thematic break) -->
<hr>

Text Formatting Elements

<p>
  <strong>Bold / important text</strong>
  <em>Italic / emphasized text</em>
  <mark>Highlighted text</mark>
  <del>Struck-through / deleted text</del>
  <ins>Underlined / inserted text</ins>
  <sub>Subscript</sub>  H<sub>2</sub>O
  <sup>Superscript</sup>  E = mc<sup>2</sup>
  <code>Inline code text</code>
  <abbr title="HyperText Markup Language">HTML</abbr>
</p>
✅
Semantic Tip

Use <strong> instead of <b> for bold, and <em> instead of <i> for italics. The semantic versions communicate meaning to screen readers and search engines — not just visual style.

Lesson 06

HTML Semantic Elements

div, section, main, article, header, footer & more

What is Semantic HTML?

Semantic elements have meaningful names that describe their purpose. Before HTML5, developers used <div> for everything. HTML5 introduced semantic tags that tell both browsers and developers what the content represents.

This matters for accessibility, SEO, and code readability.

The Core Semantic Layout Elements

<body>

  <!-- The top section of the page or a section -->
  <header>
    <h1>My Website</h1>
    <nav>
      <a href="#home">Home</a>
      <a href="#about">About</a>
    </nav>
  </header>

  <!-- The primary content of the page (only one per page) -->
  <main>

    <!-- A standalone, reusable piece of content -->
    <article>
      <h2>Blog Post Title</h2>
      <p>Article content...</p>
    </article>

    <!-- A themed grouping of content -->
    <section>
      <h2>About Me</h2>
      <p>Section content...</p>
    </section>

    <!-- Related/supplementary content -->
    <aside>
      <p>Did you know? Fun fact here.</p>
    </aside>

  </main>

  <!-- The bottom section of the page -->
  <footer>
    <p>&copy; 2025 My Website. All rights reserved.</p>
  </footer>

</body>

div vs span (Generic Containers)

<!-- div: block-level container, creates a new line -->
<div class="card">
  <p>Card content here.</p>
</div>

<!-- span: inline container, stays within text flow -->
<p>The price is <span style="color: red;">₦5,000</span> only.</p>
Element Use When…
<header> Introductory content, logos, navigation at top of page/section
<nav> A group of navigation links
<main> The central, unique content of the page (one per page)
<article> Self-contained content that could stand alone (blog post, news item)
<section> A thematic grouping of content with its own heading
<aside> Sidebar, pull quotes, related info tangential to main content
<footer> Copyright, links, contact info at bottom of page/section
<div> No semantic meaning needed — purely for grouping/styling
<span> Inline styling or grouping within text

Lesson 07

HTML Lists

ol, ul, li — and how to nest them

Unordered Lists (ul)

Use an unordered list when order doesn't matter — like a shopping list or a set of features. Each item is rendered with a bullet point. 

<ul>
  <li>Rice</li>
  <li>Beans</li>
  <li>Plantain</li>
</ul>

Ordered Lists (ol)

Use an ordered list when sequence matters — like step-by-step instructions or a ranking. Items are numbered automatically. 

<ol>
  <li>Boil water</li>
  <li>Add rice</li>
  <li>Cook for 20 minutes</li>
  <li>Serve hot</li>
</ol>

<!-- Start numbering from 5 -->
<ol start="5">
  <li>Step five</li>
</ol>

<!-- Reverse order -->
<ol reversed>
  <li>Third</li>
  <li>Second</li>
  <li>First</li>
</ol>

Description Lists (dl)

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — the language of the web.</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets — used to style HTML.</dd>
</dl>

Nested Lists

<ul>
  <li>Fruits
    <ul>
      <li>Mango</li>
      <li>Orange</li>
    </ul>
  </li>
  <li>Vegetables
    <ol>
      <li>Tomato</li>
      <li>Spinach</li>
    </ol>
  </li>
</ul>

Lesson 08

HTML Images

Displaying and optimising images on the web

The <img> Tag

Images are embedded using the self-closing <img> tag. Two attributes are mandatory: src and alt. 

<!-- Basic image from same folder -->
<img src="photo.jpg" alt="A beautiful sunset">

<!-- Image from another folder -->
<img src="images/logo.png" alt="Company Logo">

<!-- Image from the internet (URL) -->
<img src="https://example.com/photo.jpg" alt="Sample image">

<!-- Image with width, height, and title -->
<img
  src="portrait.jpg"
  alt="A portrait photo of Jane"
  width="300"
  height="400"
  title="Jane Doe — CEO"
>

Figure and Figcaption

Use <figure> and <figcaption> to semantically associate an image with its caption. 

<figure>
  <img src="eko-bridge.jpg" alt="Eko Bridge in Lagos at sunset">
  <figcaption>Eko Bridge, Lagos — photographed at dusk.</figcaption>
</figure>

Image Formats

Format Best For Supports Transparency?
JPEG / JPG Photographs, complex images No
PNG Logos, icons, images needing transparency Yes
GIF Simple animations Yes (1-bit)
SVG Icons, logos (vector — scales perfectly) Yes
WebP Modern format — smaller than JPG/PNG Yes
♿
Accessibility — Always Write Good Alt Text

The alt attribute is read aloud by screen readers for visually impaired users. Write a meaningful description. For decorative images with no meaning, use alt="" so screen readers skip them.

Lesson 09

HTML Tables

Displaying data in rows and columns

Basic Table Structure

<table>

  <!-- Table caption (optional but good for accessibility) -->
  <caption>Student Scores</caption>

  <!-- Table head -->
  <thead>
    <tr>
      <th>Name</th>
      <th>Subject</th>
      <th>Score</th>
    </tr>
  </thead>

  <!-- Table body -->
  <tbody>
    <tr>
      <td>Amara</td>
      <td>Mathematics</td>
      <td>92</td>
    </tr>
    <tr>
      <td>Chidi</td>
      <td>English</td>
      <td>85</td>
    </tr>
  </tbody>

  <!-- Table footer -->
  <tfoot>
    <tr>
      <td colspan="2">Average</td>
      <td>88.5</td>
    </tr>
  </tfoot>

</table>

Spanning Cells

<!-- colspan: merge cells horizontally -->
<td colspan="3">This cell spans 3 columns</td>

<!-- rowspan: merge cells vertically -->
<td rowspan="2">This cell spans 2 rows</td>
Tag Meaning
<table> The table container
<thead> Table header section
<tbody> Table body (main data)
<tfoot> Table footer (summaries/totals)
<tr> Table row
<th> Table header cell (bold, centred by default)
<td> Table data cell
<caption> Table title/caption
💡
Remember

HTML tables are for tabular data only. Never use tables for page layout — use semantic elements and CSS instead. This was a common mistake in early web development.

HTML Forms

Collecting input from users

The <form> Element

Forms allow users to enter data that is sent to a server. The action attribute specifies where to send the data, and method specifies how. 

<form action="/submit" method="post">
  <!-- Form fields go here -->
</form>

Input Types

<!-- Text input -->
<label for="name">Full Name:</label>
<input type="text" id="name" name="name" placeholder="Enter your name" required>

<!-- Email -->
<input type="email" id="email" name="email" placeholder="you@example.com">

<!-- Password -->
<input type="password" id="pw" name="password">

<!-- Number -->
<input type="number" min="1" max="100">

<!-- Date -->
<input type="date" name="dob">

<!-- Telephone -->
<input type="tel" name="phone">

<!-- Radio buttons -->
<input type="radio" name="gender" value="male"> Male
<input type="radio" name="gender" value="female"> Female

<!-- Checkboxes -->
<input type="checkbox" name="html" value="html"> HTML
<input type="checkbox" name="css" value="css"> CSS

<!-- File upload -->
<input type="file" accept=".jpg,.png,.pdf">

<!-- Colour picker -->
<input type="color" name="fav-color">

<!-- Range slider -->
<input type="range" min="0" max="100" step="5">

Lesson 11

HTML Video & Audio

Embedding media directly in the browser

The <video> Element

HTML5 introduced native video support — no plugins required. The <video> element embeds video directly in the browser. 

<!-- Basic video -->
<video src="movie.mp4" controls>
  Your browser does not support the video tag.
</video>

<!-- Video with all common attributes -->
<video
  width="640"
  height="360"
  controls
  autoplay
  muted
  loop
  poster="thumbnail.jpg"
  preload="metadata"
>
  <!-- Multiple sources for browser compatibility -->
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  Your browser does not support HTML5 video.
</video>

The <audio> Element

<!-- Basic audio player -->
<audio src="song.mp3" controls>
  Your browser does not support the audio element.
</audio>

<!-- Audio with multiple sources -->
<audio controls autoplay muted loop>
  <source src="song.mp3" type="audio/mpeg">
  <source src="song.ogg" type="audio/ogg">
  Your browser does not support audio playback.
</audio>

Embedding YouTube Videos

For YouTube and other hosted videos, use the <iframe> element: 

<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  title="YouTube video player"
  frameborder="0"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media"
  allowfullscreen
></iframe>
Attribute Applies To Description
controls video, audio Show play/pause/volume controls
autoplay video, audio Start playing automatically (usually muted)
muted video, audio Start with no sound
loop video, audio Repeat indefinitely
poster video Image shown before video plays
preload video, audio How much to preload: none, metadata, auto
🔇
Autoplay Policy

Most modern browsers block autoplay with sound. If you want a video to autoplay, you must also add muted. This is a user experience protection built into browsers.
