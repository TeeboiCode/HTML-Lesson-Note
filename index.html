Lesson 01

Course Introduction

What is HTML and why does it matter?

What is HTML?

HTML stands for HyperText Markup Language. It is the foundational language of the web. Every webpage you have ever visited — from Google to YouTube to Wikipedia — is built with HTML at its core.

HTML is not a programming language. It is a markup language, meaning it describes the structure and meaning of content. Think of HTML as the skeleton of a webpage, CSS as the skin and clothing, and JavaScript as the muscles that make it move.

A Brief History

HTML was created by Tim Berners-Lee in 1991. Since then it has evolved through several versions — HTML 2.0, HTML 4.01, XHTML — and the current living standard is HTML5, maintained by the WHATWG.

How the Web Works

When you type a URL into your browser:

  1. Your browser sends a request to a web server.
  2. The server responds with an HTML file.
  3. The browser reads (parses) the HTML and displays it as a page.
💡
Good to know

You don't need an internet connection to practice HTML. You can create .html files on your computer and open them directly in your browser!

Tools You Need

To write HTML, all you need is:

  • A text editor — VS Code (recommended), Notepad++, Sublime Text, or even plain Notepad.
  • A web browser — Chrome, Firefox, Edge, or Safari.

Your First HTML File

Create a file called index.html and write the following: 

<!DOCTYPE html>
<html>
  <head>
    <title>My First Page</title>
  </head>
  <body>
    Hello, World!
  </body>
</html>

Save the file and open it in your browser. You should see "Hello, World!" displayed!

Structure of an HTML Page

Anatomy of every HTML document

The Boilerplate

Every HTML page follows a standard structure called the boilerplate. Here is the full structure with explanations: 

<!-- 1. DOCTYPE declaration -->
<!DOCTYPE html>

<!-- 2. Root element -->
<html lang="en">

  <!-- 3. Head section (invisible to user) -->
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
  </head>

  <!-- 4. Body section (visible to user) -->
  <body>
    All visible content goes here
  </body>

</html>

Breaking It Down

1. DOCTYPE Declaration

<!DOCTYPE html> — This tells the browser that this document is HTML5. It must be the very first line of every HTML file. It is not a tag; it is a declaration.

2. The <html> Element

This is the root element — all other elements are nested inside it. The lang="en" attribute tells browsers and screen readers what language the page is in.

3. The <head> Element

The <head> section contains metadata — information about the document that is NOT displayed on screen. This includes the page title, character encoding, viewport settings, and links to CSS stylesheets.

4. The <body> Element

The <body> section contains everything visible on the page — text, images, links, forms, etc.

⚠️
Common Mistake

Never put visible content (text, images) inside the <head>. It belongs in the <body>. The browser will often display it anyway, but this is incorrect HTML.

HTML Comments

Comments are notes for developers that the browser ignores entirely. 

<!-- This is a comment. The browser will not display this. -->

<!-- 
  Comments can also
  span multiple lines
-->

Lesson 03

HTML Elements, Tags & Attributes

The building blocks of every webpage

What is an HTML Element?

An HTML element consists of a start tag, content, and an end tag: 

<!-- Anatomy of an element -->
<p>This is a paragraph.</p>
↑         ↑                    ↑
Start tag  Content           End tag

Opening vs Closing Tags

<!-- Opening tag -->
<p>

<!-- Closing tag (note the forward slash) -->
</p>

<!-- Self-closing / void elements (no closing tag needed) -->
<br>
<hr>
<img src="photo.jpg">
<input type="text">

Nesting Elements

Elements can be placed inside other elements. This is called nesting. Always close inner elements before outer ones. 

<!-- ✅ Correct nesting -->
<p>This is <strong>bold text</strong> inside a paragraph.</p>

<!-- ❌ Wrong nesting (overlapping tags) -->
<p>This is <strong>wrong</p></strong>

HTML Attributes

Attributes provide additional information about an element. They always go in the opening tag and follow a name="value" pattern. 

<!-- Attribute syntax -->
<tagname attribute="value">

<!-- Multiple attributes -->
<a href="https://example.com" target="_blank">Click here</a>

<!-- Boolean attributes (no value needed) -->
<input type="checkbox" checked>
<button disabled>Cannot Click</button>

Common Global Attributes

Attribute Description Example
id Unique identifier for one element id="main-title"
class Groups elements for CSS styling class="card"
style Inline CSS styling style="color: red;"
title Tooltip shown on hover title="More info"
lang Language of the element's content lang="fr"
hidden Hides the element from view hidden
✅
Best Practice

Always use lowercase for tag names and attribute names. Always wrap attribute values in double quotes. These are the modern HTML5 standards.

Lesson 04

HTML Meta Tags

Invisible data that powers your page

What Are Meta Tags?

Meta tags live inside the <head> and provide metadata — information about the page that browsers, search engines, and social media platforms use. Users don't see them directly, but they have a huge impact on how the page behaves and ranks.

Essential Meta Tags

<head>
  <!-- Character Encoding -->
  <meta charset="UTF-8">

  <!-- Responsive viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Page description (shown in Google results) -->
  <meta name="description" content="Learn HTML from scratch in this beginner course.">

  <!-- Author -->
  <meta name="author" content="Jane Doe">

  <!-- Keywords (less important today, but still used) -->
  <meta name="keywords" content="HTML, web, beginner, course">

  <!-- Refresh the page every 30 seconds -->
  <meta http-equiv="refresh" content="30">
</head>

Open Graph Meta Tags

Open Graph tags control how your page looks when shared on social media (Facebook, Twitter, WhatsApp, etc.). 

<!-- Open Graph (social media preview) -->
<meta property="og:title" content="Learn HTML – Beginner Course">
<meta property="og:description" content="A complete HTML course for beginners.">
<meta property="og:image" content="https://example.com/thumbnail.jpg">
<meta property="og:url" content="https://example.com">
Meta Tag Purpose
charset="UTF-8" Supports all characters including special symbols and emojis
name="viewport" Makes the page mobile-responsive
name="description" Search engine result snippet (keep under 160 characters)
name="author" Credits the page author
name="robots" Controls search engine indexing behavior
🔍
SEO Tip

The description meta tag is one of the most important for SEO (Search Engine Optimisation). Write it carefully — it's what users see under your page title in Google search results!

HTML Paragraphs & Headings

Structuring text content on the page

Heading Tags (h1–h6)

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important). Headings form a hierarchy — like an outline for your content. 

<h1>Main Page Title (only one per page)</h1>
<h2>Section Heading</h2>
<h3>Sub-section Heading</h3>
<h4>Sub-sub-section Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Paragraph Tags

<!-- Basic paragraph -->
<p>This is a paragraph. The browser automatically adds space before and after.</p>

<!-- Line break inside a paragraph -->
<p>First line of text.<br>
Second line within the same paragraph.</p>

<!-- Horizontal rule (thematic break) -->
<hr>

Text Formatting Elements

<p>
  <strong>Bold / important text</strong>
  <em>Italic / emphasized text</em>
  <mark>Highlighted text</mark>
  <del>Struck-through / deleted text</del>
  <ins>Underlined / inserted text</ins>
  <sub>Subscript</sub>  H<sub>2</sub>O
  <sup>Superscript</sup>  E = mc<sup>2</sup>
  <code>Inline code text</code>
  <abbr title="HyperText Markup Language">HTML</abbr>
</p>
✅
Semantic Tip

Use <strong> instead of <b> for bold, and <em> instead of <i> for italics. The semantic versions communicate meaning to screen readers and search engines — not just visual style.

Lesson 06

HTML Semantic Elements

div, section, main, article, header, footer & more

What is Semantic HTML?

Semantic elements have meaningful names that describe their purpose. Before HTML5, developers used <div> for everything. HTML5 introduced semantic tags that tell both browsers and developers what the content represents.

This matters for accessibility, SEO, and code readability.

The Core Semantic Layout Elements

<body>

  <!-- The top section of the page or a section -->
  <header>
    <h1>My Website</h1>
    <nav>
      <a href="#home">Home</a>
      <a href="#about">About</a>
    </nav>
  </header>

  <!-- The primary content of the page (only one per page) -->
  <main>

    <!-- A standalone, reusable piece of content -->
    <article>
      <h2>Blog Post Title</h2>
      <p>Article content...</p>
    </article>

    <!-- A themed grouping of content -->
    <section>
      <h2>About Me</h2>
      <p>Section content...</p>
    </section>

    <!-- Related/supplementary content -->
    <aside>
      <p>Did you know? Fun fact here.</p>
    </aside>

  </main>

  <!-- The bottom section of the page -->
  <footer>
    <p>&copy; 2025 My Website. All rights reserved.</p>
  </footer>

</body>

div vs span (Generic Containers)

<!-- div: block-level container, creates a new line -->
<div class="card">
  <p>Card content here.</p>
</div>

<!-- span: inline container, stays within text flow -->
<p>The price is <span style="color: red;">₦5,000</span> only.</p>

HTML <div> and <span> Elements

<div> and <span> are generic containers — they carry no semantic meaning on their own. They exist purely to group content so you can style or manipulate it with CSS and JavaScript.

The <div> Element

<div> is a block-level container. It starts on a new line and stretches the full width of its parent. Use it to group larger sections of content. 

<!-- Grouping a card component -->
<div class="card">
  <h2>Card Title</h2>
  <p>Card description goes here.</p>
  <a href="#">Read more</a>
</div>

<!-- Wrapping a layout column -->
<div class="column">
  <p>Left column content.</p>
</div>
<div class="column">
  <p>Right column content.</p>
</div>

The <span> Element

<span> is an inline container. It sits within a line of text without breaking the flow. Use it to style or target a specific word or phrase. 

<!-- Highlighting a word inside a paragraph -->
<p>The price is <span style="color: green; font-weight: bold;">₦5,000</span> only.</p>

<!-- Applying a class to part of a sentence -->
<p>Status: <span class="badge-active">Active</span></p>
Element Type Use For
<div> Block Grouping sections, layout wrappers, card components
<span> Inline Styling words/phrases within text, badges, labels
✅
Prefer semantic elements first

Before reaching for <div>, ask if a semantic element fits — <article>, <section>, <nav>, etc. Use <div> only when no semantic element applies.

HTML Block and Inline Elements

Every HTML element has a default display type — either block or inline. Understanding this is fundamental to controlling page layout.

Block-Level Elements

A block element always starts on a new line and takes up the full width available. Block elements stack vertically on top of each other. 

<!-- These all start on a new line -->
<div>I am a block element</div>
<p>So am I</p>
<h1>And me</h1>
<ul>Me too</ul>
<section>And me</section>

Inline Elements

An inline element does not start on a new line. It only takes up as much width as its content needs, flowing within the surrounding text. 

<!-- These stay within the text flow -->
<p>This is <strong>bold</strong>, this is <em>italic</em>, and this is a <a href="#">link</a>.</p>
<!-- span, img, input, label, code, abbr are also inline -->
Block Elements Inline Elements
<div>, <p>, <h1>–<h6> <span>, <a>, <strong>
<ul>, <ol>, <li> <em>, <img>, <input>
<section>, <article>, <header> <label>, <code>, <abbr>
<table>, <form>, <footer> <mark>, <sub>, <sup>
💡
CSS can change display type

You can override the default display with CSS: display: block, display: inline, or display: inline-block. For example, making <a> tags block-level is common for navigation buttons.

Element Use When…
<header> Introductory content, logos, navigation at top of page/section
<nav> A group of navigation links
<main> The central, unique content of the page (one per page)
<article> Self-contained content that could stand alone (blog post, news item)
<section> A thematic grouping of content with its own heading
<aside> Sidebar, pull quotes, related info tangential to main content
<footer> Copyright, links, contact info at bottom of page/section
<div> No semantic meaning needed — purely for grouping/styling
<span> Inline styling or grouping within text

HTML Lists

ol, ul, li — and how to nest them

Unordered Lists (ul)

Use an unordered list when order doesn't matter — like a shopping list or a set of features. Each item is rendered with a bullet point. 

<ul>
  <li>Rice</li>
  <li>Beans</li>
  <li>Plantain</li>
</ul>

Ordered Lists (ol)

Use an ordered list when sequence matters — like step-by-step instructions or a ranking. Items are numbered automatically. 

<ol>
  <li>Boil water</li>
  <li>Add rice</li>
  <li>Cook for 20 minutes</li>
  <li>Serve hot</li>
</ol>

<!-- Start numbering from 5 -->
<ol start="5">
  <li>Step five</li>
</ol>

<!-- Reverse order -->
<ol reversed>
  <li>Third</li>
  <li>Second</li>
  <li>First</li>
</ol>

Description Lists (dl)

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — the language of the web.</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets — used to style HTML.</dd>
</dl>

Nested Lists

<ul>
  <li>Fruits
    <ul>
      <li>Mango</li>
      <li>Orange</li>
    </ul>
  </li>
  <li>Vegetables
    <ol>
      <li>Tomato</li>
      <li>Spinach</li>
    </ol>
  </li>
</ul>

HTML Images

Displaying and optimising images on the web

The <img> Tag

Images are embedded using the self-closing <img> tag. Two attributes are mandatory: src and alt. 

<!-- Basic image from same folder -->
<img src="photo.jpg" alt="A beautiful sunset">

<!-- Image from another folder -->
<img src="images/logo.png" alt="Company Logo">

<!-- Image from the internet (URL) -->
<img src="https://example.com/photo.jpg" alt="Sample image">

<!-- Image with width, height, and title -->
<img
  src="portrait.jpg"
  alt="A portrait photo of Jane"
  width="300"
  height="400"
  title="Jane Doe — CEO"
>

Figure and Figcaption

Use <figure> and <figcaption> to semantically associate an image with its caption. 

<figure>
  <img src="eko-bridge.jpg" alt="Eko Bridge in Lagos at sunset">
  <figcaption>Eko Bridge, Lagos — photographed at dusk.</figcaption>
</figure>

Image Formats

Format Best For Supports Transparency?
JPEG / JPG Photographs, complex images No
PNG Logos, icons, images needing transparency Yes
GIF Simple animations Yes (1-bit)
SVG Icons, logos (vector — scales perfectly) Yes
WebP Modern format — smaller than JPG/PNG Yes
♿
Accessibility — Always Write Good Alt Text

The alt attribute is read aloud by screen readers for visually impaired users. Write a meaningful description. For decorative images with no meaning, use alt="" so screen readers skip them.

HTML Tables

Displaying data in rows and columns

What is an HTML Table?

An HTML table is used to display data arranged in rows and columns — like a spreadsheet. Tables are ideal for structured, relational data such as schedules, scores, pricing plans, and comparison charts.

A table is built from the outside in: first the <table> wrapper, then semantic sections (<thead>, <tbody>, <tfoot>), then rows (<tr>), and finally individual cells (<th> or <td>).

Key Definitions

<table>
The outer container that wraps the entire table.
<thead>
The header section — holds the column label row(s). Browsers may repeat this when printing long tables.
<tbody>
The body section — holds all the main data rows.
<tfoot>
The footer section — typically holds totals, averages, or summary rows.
<tr> (Table Row)
Defines a single horizontal row inside any table section.
<th> (Table Header cell)
A header cell — bold and centred by default. Use inside <thead> to label columns.
<td> (Table Data cell)
A regular data cell — left-aligned by default. Used inside <tbody> and <tfoot>.
<caption>

Lesson 10

HTML Forms

Collecting input from users

What is an HTML Form?

An HTML form is a section of a webpage that collects input from the user and sends it to a server for processing. Forms are used everywhere — login pages, registration, search bars, checkout flows, contact pages, and surveys.

Every form is built around the <form> element, which acts as a container. Inside it you place various input controls — text fields, dropdowns, checkboxes, radio buttons, and buttons.

Key Definitions

<form>
The container element for all form controls. Its action attribute sets the URL that receives the data, and method sets how it is sent (GET or POST).
<input>
A self-closing element that creates many types of form controls depending on its type attribute — text, email, password, checkbox, radio, file, date, and more.
<label>
Provides a visible, clickable description for an input. Linked to an input via matching for and id values. Essential for accessibility.
<textarea>
A multi-line text input. Unlike <input>, it has an opening and closing tag. Use rows and cols to set its size.
<select> and <option>
Creates a dropdown menu. Each choice is an <option> element inside the <select>.
<button>
A clickable button. type="submit" submits the form, type="reset" clears all fields, type="button" does nothing by default (used with JavaScript).
<fieldset> and <legend>
<fieldset> groups related form controls with a visible border. <legend> provides a title for the group. Both improve accessibility and organisation.
required
A boolean attribute that prevents form submission if the field is empty. The browser shows a built-in validation message.
placeholder
Hint text shown inside an input when it is empty. It disappears when the user starts typing. Do not use it as a replacement for a <label>.

The <form> Element

Forms allow users to enter data that is sent to a server. The action attribute specifies where to send the data, and method specifies how. 

<form action="/submit" method="post">
  <!-- Form fields go here -->
</form>

  novalidate
>

enctype Encoding type — use multipart/form-data for file uploads enctype="multipart/form-data"

novalidate Disables HTML5 built-in validation on submission novalidate

✅
Always link labels to inputs

Use <label for="id"> matched to <input id="id">. This lets users click the label text to focus the input — critical for keyboard and screen reader users. Never skip labels.

⚠️

HTML5 validation only runs in the browser. A user can bypass it by disabling JavaScript or sending a direct HTTP request. Always validate data on the server side as well.

Lesson 11

HTML Video & Audio

Embedding media directly in the browser

What is HTML Media?

Before HTML5, embedding video or audio on a webpage required third-party plugins like Adobe Flash. HTML5 introduced native <video> and <audio> elements that work directly in the browser — no plugins needed.

You can either host media files yourself (self-hosted) or embed content from platforms like YouTube using an <iframe>.

Key Definitions

<video>
Embeds a video player in the page. Supports MP4, WebM, and Ogg formats. Requires a src attribute or nested <source> elements.
<audio>
Embeds an audio player. Supports MP3, WAV, and Ogg formats. Works the same way as <video> but without a visual frame.
<source>
A self-closing element nested inside <video> or <audio> to provide multiple file formats. The browser picks the first format it supports.
controls
A boolean attribute that shows the browser's built-in play/pause/volume/seek controls. Without it, the media plays with no user interface.
autoplay
Starts playing the media as soon as the page loads. Most browsers block autoplay with sound — combine with muted for it to work reliably.
muted
Starts the media with the volume set to zero. Required for autoplay to work in most browsers.
loop
Makes the media restart automatically when it reaches the end.
poster
An image URL shown as a thumbnail before the video starts playing. Only applies to <video>.
preload
Hints to the browser how much to load before the user presses play. Values: none (load nothing), metadata (load duration/dimensions only), auto (load the whole file).
<iframe>
An inline frame that embeds another webpage inside the current one. Used to embed YouTube, Vimeo, and Google Maps. The src attribute holds the embed URL.
<track>
Adds subtitles, captions, or chapter markers to a video. Uses WebVTT (.vtt) files. The kind attribute specifies the type (subtitles, captions, descriptions).

The <video> Element

HTML5 introduced native video support — no plugins required. The <video> element embeds video directly in the browser. 

<!-- Basic video -->
<video src="movie.mp4" controls>

</video>

  Your browser does not support HTML5 video.
</video>

  Your browser does not support the audio element.

1. Which attribute shows the browser's built-in play/pause controls on a video?

✅ Correct Answer: B — controls

Definition: The controls boolean attribute tells the browser to display its native media player UI — including play/pause, a seek bar, volume control, and fullscreen button. Without it, the video plays (or doesn't) with no visible interface.

2. Why do we provide multiple <source> elements inside a <video> tag?

✅ Correct Answer: C — To provide fallback formats

Definition: Not all browsers support the same video formats. By listing multiple <source> elements (e.g., MP4 and WebM), the browser tries each in order and plays the first one it can handle. This ensures cross-browser compatibility.

3. What does the poster attribute do on a <video> element?

✅ Correct Answer: B — Shows a thumbnail image before the video plays

Definition: poster="image.jpg" specifies an image to display in the video frame before the user presses play (or while the video is loading). It improves the visual appearance of the page and gives users a preview of the content.

4. Why must autoplay usually be combined with muted?

✅ Correct Answer: B — Browsers block autoplay with sound to protect users

Definition: Modern browsers (Chrome, Firefox, Safari) enforce an autoplay policy that blocks media from playing with sound automatically, as unexpected audio is disruptive. Adding muted bypasses this restriction, allowing the video to autoplay silently — common for background hero videos.

5. What is the difference between using a <video> tag and an <iframe> for embedding video?

✅ Correct Answer: B — <video> for self-hosted files; <iframe> for external platforms

Definition: <video> plays media files (MP4, WebM) that you host on your own server. <iframe> embeds an entire external webpage — like a YouTube player — inside your page. YouTube videos cannot be played with <video> directly; you must use their embed URL in an <iframe>.
